SocketReceive(topic: String, event: Either<SocketSystemEvent, SocketCustomEvent>, ref: String? = null, data: T? = null, version: String, success: Boolean, error: APIError? = null)

A model for receiving the replied object from the eWallet web socket API. It closely follows the usual envelope used in the eWallet HTTP APIs with some added attributes related to web socket. Note that those events can either be successful or not.

For example, when sending the SocketCustomEvent.TRANSACTION_CONSUMPTION_FINALIZED, it is possible to receive the finalized consumption OR an error stating that it was finalized in a failed state because there were not enough funds for the actual transaction to proceed.

Parameters

topic - The topic (channel) to which the event was sent (probably the name of the channel you joined).

event - The event could be either SocketSystemEvent or SocketCustomEvent.

ref - null for events emitted from the server in response to a server action.

data - The data relevant to the event. Could be null if success field is equal to false (but could also contain the SocketReceive.Data to provide context for the error).

version - The web socket API version.

success - Defines if the event is the result of a successful action or not.

error - The error resulting from the action generating the event
