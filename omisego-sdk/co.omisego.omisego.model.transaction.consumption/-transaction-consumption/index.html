TransactionConsumption

data class TransactionConsumption : Parcelable, Listenable<TransactionConsumptionListener>, SocketData

Represents a transaction consumption

Parameters

id - The unique identifier of the consumption

status - The status of the consumption (pending, confirmed or failed)

amount - The amount of token to transfer (down to subunit to unit)

estimatedRequestAmount - The estimated amount in the request currency

estimatedConsumptionAmount - The estimated amount in the consumption currency

finalizedRequestAmount - The final amount to be transferred after exchange in the request currency

finalizedConsumptionAmount - The final amount to be transferred after exchange in the consumption currency

token - The token for the request In the case of a type "send", this will be the token that the consumer will receive In the case of a type "receive" this will be the token that the consumer will send

correlationId - An id that can uniquely identify a transaction. Typically an order id from a provider.

idempotencyToken - The idempotency token of the consumption

transaction - The transaction generated by this consumption (this will be null until the consumption is confirmed)

address - The address used for the consumption

user - The user that initiated the consumption

account - The account that initiated the consumption

transactionRequest - The transaction request to be consumed

socketTopic - The topic which can be listened in order to receive events regarding this consumption

createdAt - The creation date of the consumption

expirationDate - The date when the consumption will expire

approvedAt - The date when the consumption got approved

rejectedAt - The date when the consumption got rejected

confirmedAt - The date when the consumption got confirmed

failedAt - The date when the consumption failed

expiredAt - The date when the consumption expired

metadata - Additional metadata for the consumption

encryptedMetadata - Additional encrypted metadata for the consumption
