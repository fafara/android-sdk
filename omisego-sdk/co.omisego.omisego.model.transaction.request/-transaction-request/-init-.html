TransactionRequest(id: String, type: TransactionRequestType, token: Token, amount: BigDecimal?, address: String?, user: User?, account: Account?, correlationId: String?, socketTopic: SocketTopic<TransactionRequestListener>, maxConsumptions: Int?, status: TransactionRequestStatus, allowAmountOverride: Boolean, maxConsumptionsPerUser: Int?, requireConfirmation: Boolean, expirationDate: Date?, expirationReason: String?, consumptionLifetime: Int?, createdAt: Date?, expiredAt: Date?, formattedId: String, metadata: Map<String, Any>, encryptedMetadata: Map<String, Any>)

Represents a transaction request

Parameters

id - The unique identifier of the request

type - The type of the request (send of receive)

token - The token for the request In the case of a type "send", this will be the token taken from the requester In the case of a type "receive" this will be the token received by the requester

amount - The amount of token to use for the transaction (down to subunit to unit) This amount needs to be either specified by the requester or the consumer

address - The address from which to send or receive the tokens

user - The user that initiated the request

account - The account that initiated the request

correlationId - An id that can uniquely identify a transaction. Typically an order id from a provider

socketTopic - The topic which can be listened in order to receive events regarding this request

maxConsumptions - The maximum number of time that this request can be consumed. Default null (unlimited).

status - The status of the request (valid or expired)

allowAmountOverride - Allow or not the consumer to override the amount specified in the request Note that if amount is nil and allowAmountOverride is false the init will fail and return null.

maxConsumptionsPerUser - The maximum number of consumptions allowed per unique user. Default null (unlimited).

requireConfirmation - A boolean indicating if the request needs a confirmation from the requester before being proceeded

expirationDate - The date when the request will expire and not be consumable anymore. Default null (never expired).

expirationReason - The reason why the request expired

consumptionLifetime - The amount of time in millisecond during which a consumption is valid. Default null (forever).

createdAt - The creation date of the request

expiredAt - The date when the request expired

formattedId - An id that can be encoded in a QR code and be used to retrieve the request later

metadata - Additional metadata for the transaction request

encryptedMetadata - Additional encrypted metadata for the transaction request
